# GEMINI.md

This file provides guidance to Gemini when working with code in this repository.

## Repository Purpose

Centralized hub for AI agent definitions and reusable skills across Claude, OpenCode, and Codex. Agent source files live in `src/agents/` (pure markdown, no frontmatter). Platform-specific output is generated by the build system — do not edit generated files directly.

## Build System

```bash
pnpm build          # Generate platform output from src/ + agents.config.json
pnpm sync           # Build + deploy to local tool dirs (~/.claude, ~/.codex, ~/.config/opencode)
pnpm test           # Run vitest tests
pnpm lint           # Run ESLint
```

- Agent metadata (name, description, color, model, temperature) is defined in `agents.config.json` with per-platform overrides.
- To add a new agent: create `src/agents/{name}.md` + add entry to `agents.config.json`.
- To add a new platform: add entry to `agents.config.json` under `platforms`.

## Default Orchestration

`it-task-master` is the default orchestrator. Start with it for general or mixed requests — it determines task type and delegates to the correct specialist agent.

## Tool/Library Documentation Verification (Context7 MCP)

When working with a specific tool, library, or framework, first check whether documentation is available via Context7 MCP. If available, use Context7 docs to verify APIs and usage patterns.

Practical flow: resolve library ID -> query docs -> implement.

## UI Design & Prototyping (Stitch MCP)

Stitch MCP is available for UI design and prototyping workflows: create/list projects, generate screens from text prompts, edit screens, generate design variants, and retrieve project/screen details.
